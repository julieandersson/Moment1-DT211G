Arbetsprocess



Nytt npm-projekt


För att lösa denna uppgift började jag med att skapa en ny katalog på min dator som jag döpte till "Moment 1". I katalogen gick jag in i kommandoterminalen och angav npm init för att helt enkelt initiera ett nytt projekt. Efter detta öppnade jag upp katalogen i VsCode. Här försökte jag med kortkommandot code . men det fungerade inte för mig, så jag öppnade den direkt i VsCode istället.



Installation av Parcel


Efter att ha öppnat upp katalogen i VsCode fann jag där en package.json-fil, som hade skapats när jag initierade det nya projektet. Jag gick in i terminalen direkt i VsCode och installerade Parcel med kommandot npm install parcel --save-dev, för att spara paketet i utvecklings-läge.

När Parcel var färdiginstallerat skapades det en mapp vid namn node_modules/ som jag valde att lägga in i en .gitignore-fil som jag skapade, för att inte versionshantera denna fil. Detta för att node_modules/ innehåller alla externa beroenden och dess filer som krävs för att köra projektet. Eftersom denna mapp är mycket stor skulle det innebära onödig uppladdning och lagring av stora mängder filer.
Jag placerade även dist och parcel-cache-mapparna i .gitignore-filen när de hade skapats senare.

Skapandet av webbplatsen


Innan jag satte igång och koda min webbplats skapade jag en src-mapp där samtlig HTML, CSS, JavaScript och bilder placerades i vardera mapp. Efter detta skapade jag ett git-repo för projektet där jag även skapade en extra branch vid namn "development", som en utvecklingsbranch istället för att göra alla ändringar i mainbranchen som senare blir publicerings-branchen.
För att köra igång med repot i VsCode skrev jag kommandona git pull och git checkout development i terminalen och därefter satte jag igång med skapandet av kod för webbplatsen. Under utvecklingen använde jag kommando npm run start för att starta live server där jag kan se alla ändringar jag gör. Jag gjorde även regelbundna commits så att alla ändringar sparades, med kommandona git add *, git commit -m "meddelande" och git push.



Publicering av webbplatsen


När jag var färdig med webbplatsen slog jag samman utvecklings-branchen "development" med publicerings-branchen "main" genom kommandona git checkout main och git merge development. Därefter öppnade jag mitt projekt i Netlify, gick till "add new site" och deploy with github. Därefter sökte jag upp mitt repo och skapade mina sista inställningar innan jag publicerade webbplatsen via Netlify.



Automatisering / Verktyg


Automatiseringen som gjorts är processer som omvandling och optimering av grafikfiler, säkerställande av bakåtkompatibilitet, minimering av CSS- och HTML-filer, omvandling och minimering av JavaScript-filer samt kopiering av arbetsfiler till mappen för publicering.
Majoriteten av detta sker automatiskt under utvecklingens gång med hjälp utav NodeJs med npm-paketet Parcel.

Verktygen jag använt för denna uppgft är Visual Studio Code tillsammans med kommandoterminalen, Github och Netlify samt studiematerialet för Moment 1 i kursen DT211G, såsom teori & läsanvisningar samt föreläsningar.
